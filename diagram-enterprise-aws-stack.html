Key Architectural Decisions:

1. EntraID → Cognito Federation: Azure EntraID remains the enterprise SSO source of truth. Cognito acts as the AWS-side token service, federating EntraID identities via SAML 2.0. A pre-token-generation Lambda enriches Cognito JWTs with the same role, assigned_account_ids, and compliance_clearance claims the platform SDK expects — no agent code changes required.

2. Cisco Cloud APIC → AWS VPC Integration: Cisco Cloud APIC deploys a CSR (Cloud Services Router) pair in the AWS VPC and extends the on-prem ACI fabric using VXLAN tunnels over Transit Gateway. This gives the enterprise consistent EPG-based microsegmentation policies across on-prem data center and AWS workloads. Agent-to-MCP-to-DB traffic is governed by ACI contracts, providing L4–L7 policy enforcement that complements OPA's application-layer authorization.

3. Fargate over EKS: Fargate eliminates Kubernetes node management overhead. Each agent and MCP server runs as an ECS service with its own task definition, auto-scaling rules, and IAM task role. For enterprises that prefer K8s, the same Docker images deploy to EKS with Fargate launch type — the agent code is container-runtime agnostic.

4. Bedrock as Unified LLM Layer: All LLM calls route through AWS Bedrock, providing a single billing surface, IAM-based model access control, built-in guardrails, and invocation logging. LiteLLM's Bedrock provider translates the existing model_name config to Bedrock model IDs — the only change is the environment variable (bedrock/anthropic.claude-3-5-haiku-20241022 vs gpt-4o-mini).

5. RAG with pgvector on RDS: Rather than adding a separate vector database, RAG embeddings are stored in the existing RDS PostgreSQL instance using pgvector. This keeps the data layer simple (one database engine), leverages existing RDS backup/HA, and allows SQL JOINs between vector search results and relational CRM/payment data in a single query.
